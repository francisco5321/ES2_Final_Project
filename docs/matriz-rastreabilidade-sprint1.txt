Matriz de Rastreabilidade - Sprint 1
GREENHERB
Tema: Autenticacao e controlo de acesso


Resumo

- Area funcional: autenticacao e controlo de acesso
- Endpoints cobertos:
  POST /auth/register
  POST /auth/login
  POST /auth/refresh
  GET /users/me
- Testes unitarios implementados: 11
- Testes de integracao implementados: 5
- Testes de sistema implementados: 3
- Tecnicas usadas:
  Particionamento de Equivalencia
  Fluxo nominal
  Validacao de token JWT
  Tratamento de erro interno
- Codigos HTTP documentados:
  200, 201, 400, 401, 409 e 500


Testes Unitarios

TU-01
Regra: RN-AUTH-01 - Registo de utilizador com dados validos
Endpoint/componente: POST /auth/register
Tecnica: Particionamento de Equivalencia
Codigo HTTP esperado: 201
Resultado esperado: o utilizador e criado com sucesso e sao devolvidos accessToken e refreshToken.
Pre-condicoes: nenhuma; repositorio em memoria limpo.

TU-02
Regra: RN-AUTH-02 - Email repetido deve ser rejeitado
Endpoint/componente: POST /auth/register
Tecnica: Particionamento de Equivalencia
Codigo HTTP esperado: 409
Resultado esperado: a operacao falha com "User already exists.".
Pre-condicoes: ja existe um utilizador registado com o mesmo email.

TU-03

Tecnica: Particionamento de Equivalencia

Pre-condicoes: nenhuma.

TS-02

Codigo HTTP esperado: 201, 200 e 200
Resultado esperado: o utilizador obtem novos tokens atraves do refresh e continua a aceder ao endpoint protegido.

TS-03

Observacoes

- A numeracao segue o padrao do enunciado: TU para unidade, TI para integracao e TS para sistema.
- Os testes unitarios focam-se na logica de autenticacao.
- Os testes de integracao validam a interacao entre rotas, middleware e servicos.
- Os testes de sistema foram implementados como fluxos end-to-end da autenticacao.
